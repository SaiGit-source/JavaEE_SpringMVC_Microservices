Overview of the full-stack and MVC

MVC = Model-View-Controller

Model is where your data resides, where you define schemas, and models for your data
View is the visuals (HTML). Pure HTML of the application.  
Controller is where you have the logic of the application. The functions that make your application run.

Model will be handles by Spring
Controller => Spring. Some logic could be handled by React as well
View => React


Why HTTP works without a controller
Because this backend uses Spring Data REST.

Key points
pom.xml includes:

spring-boot-starter-data-jpa
spring-boot-starter-data-rest
com.crm.crmDemo.repo.contactRepo extends JpaRepository<Contact, Long>

Spring Data REST auto-exposes repository interfaces as REST endpoints

No @RestController is needed
No manual controller class is required
